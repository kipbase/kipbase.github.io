CSS伪类伪元素选择器

发表于 2016-05-09 | 分类于 CSS

CSS中选择器是最基础的内容，也是使用中最常用的内容，如何掌握选择器的使用，理解各种选择器的意义，控制选择器的优先级，尤为重要。

选择器的分类

CSS中选择器大致分为以下几类：

  • 元素选择器：通过直接选择HTML中的元素进行选择。
  • 类与ID选择器：通过HTML元素中的class属性和ID属性进行选择。
  • 属性选择器：对HTML元素的属性进行选择，包括class属性与ID属性。
  • 文档结构选择器：通过元素之间的结构关系进行选择，如父子关系、邻居关系等。
  • 伪类和伪元素选择器：对文档中不存在的结构和某些元素的状态进行选择。也是本文着重说明的选择器。

伪类与伪元素选择器

CSS中伪类和伪元素选择器经常会产生混淆，但这两个概念还是有一些区别的，虽然现在来说这两种选择器用起来几乎一样，当时本着严谨的学习态度，弄清这两种选择器的区别还是非常有必要的。

伪类选择器

从名字上可以理解，伪类选择器就是对文档中已经存在的元素假如某些”类”，如最常见的


此选择器的意思是当鼠标放到a元素上，对元素的样式进行修改，这就相当于在鼠标移入时，为a元素添加hover类

从此定义可以看出来，伪类选择器更多的选择一种”状态”，如:hover就是选择鼠标在上面的状态，:visited就是选择已经访问过的状态。

伪元素选择器

与伪类选择器不同，伪元素顾名思义，就是再HTML中加入假想的元素，如


假如p元素的内容是一段文字，:first-letter选择器的作用就是将这段文字的第一个字母独立出来，成为一个单独的元素，从而对其进行样式的操作

从此可以看出来，伪元素不同于伪类，伪元素选择器就是创建文档中所没有的元素，如:before就是在前面加入一个新元素等。

异同

在实际使用中，只要弄清选择器的意义，伪类和伪元素选择器的使用并没有什么不同，因此如果不理解本质区别并不会有什么影响。
要注意到的是，在CSS3中为了将这两种进行区别，伪元素选择器使用了两个冒号，如::before，但是由于老版本IE的问题（老版本IE只支持单冒号的伪元素选择器），因此对于大部分人都已经养成了用单冒号的习惯，并且现代浏览器对于单双冒号的识别没有区别，因此更加建议这两种都使用单冒号。
