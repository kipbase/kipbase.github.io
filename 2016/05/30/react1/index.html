React技术栈快速上手1——React

发表于 2016-05-30 | 分类于 React

配套代码：https://github.com/kipbase/react-stack-build
学习基础：少量es6语法、node.js
React是现在非常流行的一个框架，发展迅速，在此基础上发展出各种框架与功能，如redux、react-router等等，形成了一个完善但又复杂技术栈，初次接触React，一定会因为处理各种框架之间的关系感到困惑，此系列名为“React快速上手”，旨在对React技术栈中最常用的一些框架等进行总结与学习，并理清各个框架之间的关系，快速上手React工程。
React技术栈的核心就是React，首先总结React相关内容，对以后各个模块的理解有着重要的作用。

编译器Babel

为了将JSX语法编译成通用的JS代码，因此要使用编译器对其进行编译，React在早期，开发了一套名为JSTransform，目前已经放弃维护，因此现在所使用的编译器名为Babel，非常完善，提供了更多的功能，在实际的编写程序中，JSX语法与ES6语法都需要Babel进行编译处理。

  1. 最为简单的Babel转换方式为使用命令行转码，在全局中安装babel-cli：
    npm install --global babel-cli
    从而使用babel命令进行转换。
    使用babel命令转换比较简单，并且在实际构建中较少使用到，因此使用方法请查阅官方文档。
  2. 在babel-cli中还有一个命令babel-node，此命令可以直接运行ES6代码，因此在开发过程中会经常使用。
  3. 如果想调用babel的API进行转换，那么需要在项目中安装babel-core包，从而使用babel提供的API进行转换，该命令一般在生成最终项目文件时（非开发）进行使用
    在实际的开发中，babel的使用一般会被配置在自动化构建工具中，如现在最为常用的webpack，因此具体的使用在后面总结webpack是进行学习。

注意事项

  1. 对于React组件来说，每一个组件只能有一个唯一的父元素；这从上面的简介可以看出，JSX组件所生成的对象有一个type属性，而这个属性的值只能是唯一的，如果想要在一个组件中渲染多个元素，那么就应该用div等元素将其包裹起来。
  2. 如果JSX标签是闭合式的，那么就必须要在结尾用/>，不可以像HTML一样省略。
